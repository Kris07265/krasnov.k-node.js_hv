Homework 16 "Генерація HTML. Шаблонізатори"

Завдання 1: Підключення шаблонізатора
Оберіть один шаблонізатор із розглянутих на уроці — EJS, Pug або Handlebars — і підключіть його до сервера.

Налаштуйте шаблонізатор у server.js (три знайомі з уроку кроки: view engine, папка views, і далі res.render у маршрутах).

Очікуваний результат: сервер запускається без помилок.

Завдання 2: Сторінка з динамічними даними
Створіть маршрут GET /profile, який рендерить шаблон profile із такими даними:

javascript

Copy code
res.render('profile', {
name: 'Ваше ім'я',
role: 'Студент курсу Node.js',
hobbies: ['хобі 1', 'хобі 2', 'хобі 3']
});
Шаблон повинен вивести:

<h1> з ім'ям;
<p> з роллю;
список <ul> з хобі, згенерований циклом — не хардкодьте три <li> руками!
Як робити цикл у вашому шаблонізаторі, доведеться загуглити, бо це частина завдання. Підказки для пошуку: «ejs loop array», «pug each», «handlebars each».

Очікуваний результат: сторінка http://localhost:3000/profile показує ім'я, роль і список хобі. Перевірка, що цикл справжній: додайте четвертий елемент у масив — на сторінці має автоматично з'явитися четвертий <li> без жодної зміни шаблону.

Завдання 3: Сторінка 404 через шаблон
Замініть текстову відповідь 404 на рендер шаблону not-found, який виводить:

заголовок «404»;
повідомлення з URL, який намагався відкрити користувач (він є в req.url).
Статус відповіді має залишитися 404:

javascript

Copy code
res.status(404).render('not-found', { url: req.url });
Перевірте:

bash

Copy code
curl -I http://localhost:3000/abrakadabra
Очікуваний результат: у браузері http://localhost:3000/abrakadabra показує HTML-сторінку з текстом на кшталт «Сторінки /abrakadabra не існує», а curl підтверджує код HTTP/1.1 404 Not Found.

Завдання 4: Умова в шаблоні
Розширте маршрут /profile: додайте в дані поле isOnline (значення true або false). У шаблоні виведіть:

якщо isOnline === true — текст Онлайн;
інакше — Офлайн.
Синтаксис умов у вашому шаблонізаторі — знову у документацію

Очікуваний результат: зміна значення isOnline у server.js змінює текст на сторінці без зміни шаблону.

Завдання 5: Розширення (обов'язкове)
Створіть маршрут GET /students, який рендерить шаблон зі списком студентів. Дані — масив об'єктів:

javascript

Copy code
const students = [
{ name: 'Оля', level: 'junior' },
{ name: 'Максим', level: 'middle' },
{ name: 'Ірина', level: 'senior' }
];
Шаблон має вивести HTML-таблицю: колонки «Ім'я» та «Рівень», по рядку на кожного студента — циклом.

Очікуваний результат: таблиця на http://localhost:3000/students, кількість рядків відповідає довжині масиву.
